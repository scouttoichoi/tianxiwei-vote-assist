1
Create a new instance

Click the white button Add Instance at the top right of the left panel.

2
Configure Name & Proxy

Enter an easy-to-remember name (e.g., Auto-Signup - 1) and paste your proxy address (format: ip:port or user:pass@ip:port). Click Save.

3
Edit or Delete instances

Click on any instance in the table list to highlight it. You can click the Pencil icon to edit proxy configurations, or click the Trash can icon to permanently delete the instance.

⚠️
Proxy Allocation Warning If running more than 3 instances concurrently, assign a dedicated private proxy to each instance. Running multiple accounts under your home Wi-Fi high-frequency IP will trigger a Bugs IP block and lock your local network temporarily.

Auto IP-Melt Safety Fusion and Anti-Ban Strategy

The voting assistant is equipped with an integrated real-time monitor to safeguard your local IP network from getting permanently banned.

Bugs employs a highly strict anti-spam policy. High-frequency registration requests originating from a single IP will prompt an immediate network block. To prevent mass account bans, the app actively parses stdout console logs. The moment any multi-instance browser thread intercepts a Bugs response containing the warning message: `Your IP is temporarily unavailable for membership`, it immediately initiates a **System-Wide Emergency Melt**: terminating all active browser instances, shutting down tasks, and displaying an desktop alert reminding you to restart your router/Wi-Fi modem or change VPN servers before resuming.

Quick Troubleshooting Guide for Beginners

IP Block Error: "Your IP is temporarily unavailable..."

Cause: Your home internet IP has been temporarily blacklisted by Bugs due to high-frequency registrations.
Solution: Power off your home Wi-Fi router/modem for 30 seconds, then plug it back in to acquire a fresh new dynamic IP from your ISP, or turn on a trusted VPN on your computer before running the app.

Ad farming warns: "No active emulator found"

Cause: The ADB daemon on your emulator is disabled, or a firewall/security software is blocking port mapping.
Solution: Close the emulator and restart it. Go to Emulator Settings ➔ Developer Options ➔ toggle **USB Debugging** OFF and back ON to refresh the communication bridge.
